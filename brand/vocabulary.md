# Webflow Vocabulary Reference

## Preferred Terms

Words and phrases that reflect the Webflow voice.

### Action Verbs
Use strong, specific verbs. These appear frequently across Webflow surfaces:

| Use | Instead of |
|-----|-----------|
| build | create, make |
| design | style, prettify |
| publish | deploy, ship, push live |
| accelerate | speed up, turbocharge |
| scale | grow, expand |
| streamline | simplify, make easier |
| deliver | provide, give |
| unlock | enable, allow |
| craft | produce, generate |

### Empowerment Language
Webflow frames its value as giving power to people:

- "superpowers" — used deliberately, not casually
- "control" — especially around data, AI, and publishing
- "autonomy" / "autonomous" — teams acting independently
- "guardrails" — safety without restriction
- "freedom to move" — agency without chaos

### Technical Terms (Use Accurately)
- **CMS** — not "content management system" in marketing copy; spell out only in docs if needed
- **Designer** — the Webflow visual editor (capitalized as a product name)
- **Editor** — the content editing interface (distinct from Designer)
- **Interactions** — Webflow's animation system (capitalized as feature name)
- **Logic** — Webflow's conditional visibility system
- **Localization** — not "translation" (broader scope)

### Enterprise Vocabulary
These terms signal maturity to enterprise buyers:

- enterprise-grade
- unified
- extensible
- governance
- compliance
- at scale
- cross-functional

## Words to Avoid

| Avoid | Why | Use instead |
|-------|-----|-------------|
| revolutionary | Hyperbole without proof | (state the specific outcome) |
| game-changing | Overused, vague | (state the specific outcome) |
| simple / easy | Can feel dismissive of real complexity | clear, intuitive, streamlined |
| just | Minimizes effort ("just add a class") | (remove it) |
| utilize | Pretentious synonym for "use" | use |
| leverage | Corporate jargon | use, apply, build on |
| synergy | Meaningless | collaboration, integration |
| best-in-class | Unprovable | (state specific differentiator) |
| cutting-edge | Vague | (describe the actual capability) |
| seamless | Overused to the point of invisibility | (describe what actually happens) |
| please note that | Filler | (just state the thing) |
| it should be noted | Passive filler | (just state the thing) |
| in order to | Wordy | to |

## Capitalization

- **Webflow** — always capitalized, never "webflow" or "WebFlow"
- **Designer**, **Editor**, **CMS**, **Logic**, **Interactions** — capitalized when referring to Webflow product features
- **Enterprise** — capitalized when referring to the plan tier
- Page names and feature names follow title case in UI, sentence case in body copy

## Numbers & Metrics

- Use numerals for metrics and results: "332% ROI," "7x faster," "60 requests/minute"
- Spell out numbers under 10 in editorial prose, except in data-heavy contexts
- Always include the unit: "18% increase in engagement" not "18% increase"
- Lead with the number when it's the point of the sentence
